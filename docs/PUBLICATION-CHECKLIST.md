# Publication Checklist — Portfolio técnico

Checklist previo a publicar el repositorio o enlazarlo desde LinkedIn/CV.

## Revisión de privacidad

- [ ] Revisar que la documentación pública no contiene datos personales.
- [ ] Revisar que no aparecen cuentas técnicas concretas si no aportan valor explicativo.
- [ ] Revisar capturas antes de hacer público el repositorio.
- [ ] Mantener los documentos Word completos como memoria privada.

## Revisión de documentación

- [ ] Verificar que las imágenes cargan correctamente en GitHub.
- [ ] Verificar que no quedan textos como `pendiente`, `en curso` o `arrancar laboratorio` en laboratorios cerrados.
- [ ] Verificar que el README principal está orientado a recruiters.
- [ ] Verificar coherencia con LinkedIn/CV.
- [ ] Confirmar que la presentación pública es la de portfolio técnico profesional.

## Revisión técnica

- [ ] Comprobar que los enlaces relativos funcionan.
- [ ] Comprobar que los laboratorios cerrados figuran como `Completado v1`.
- [ ] Revisar que las decisiones técnicas importantes están explicadas.
- [ ] Revisar que las incidencias reales aparecen como troubleshooting y no como fallos sin resolver.

## Revisión final

Antes de cambiar la visibilidad del repositorio, realizar una última búsqueda local de información privada, datos de contacto no deseados y referencias internas que no sean necesarias para entender los laboratorios.

Si aparece información que no debe publicarse, corregirla antes de cambiar la visibilidad del repositorio a público.
